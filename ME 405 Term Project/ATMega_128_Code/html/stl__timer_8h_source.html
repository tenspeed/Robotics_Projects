50 /** This union holds a 32-bit time count. The count can be accessed as a single 32-bit
51  * number, as two 16-bit integers placed together, or as an array of 8-bit characters.
52  * This is helpful because the time measurement we use consists of the number in a
53  * 16-bit hardware counter and a 16-bit overflow count which should be put together
54  * into one time number. The characters are handy for looking at individual bits.

66 /** This class holds a time stamp which is used to measure the passage of real time in
67  * the world around an AVR processor. This version of the time stamp implements a
68  * 32-bit time counter that runs at one megahertz. There's a 16-bit number which is
69  * copied directly from a 16-bit hardware counter and another 16-bit number which is
70  * incremented every time the hardware counter overflows; the combination of the two
71  * is a 32-bit time measurement. The time stamp is to be interpreted and printed as
72  * containing a whole number of seconds and a number of microseconds. The conversion

146 /** This class implements a timer to synchronize the operation of tasks on an AVR. The
147  * timer is implemented as a combination of a 16-bit hardware timer (Timer 1 is the
148  * usual choice) and a 16-bit overflow counter. The two timers' data is combined to
149  * produce a 32-bit time count which is used to decide when tasks run. WARNING: This
150  * timer does not keep track of the time of day, and it overflows after a little more
151  * than an hour of use. Another version of the stl_timer can be used when longer time
152  * periods need to be kept track of, to lower precision.
153  */
